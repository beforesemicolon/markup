Effect helper

When using state you will noticethat only the part of the template dependent on the state willupdate with changes.

For those situations you would want to perform "side effects" on thetemplate, meaning, you want other part of the template to changewith some state change.

That's when the effect helper comes in. To help youperform these side effects.

const counter = html`<p>${count}</p><p>${effect(count, () => count() * 2)}</p>
`;

The above example shows a simple example of performing a sideeffect. The bellow example show using effect we canreact to the activeTabstate change to get the content and render it.

type Tab = 'home' | 'about' | 'contact';const tabContent: Record<Tab, HTMLTemplate> = {home: html`<h1>Welcome</h1>`,contact: html`<p>Tell us what you think</p>`,about: html`<p>We are here to help you</p>`,
}const [activeTab, setActiveTab] = state<Tab>("home");const app = html`<h1>Tab Example</h1><ul><liattr.class.active="${is(activeTab, 'home')}"onclick="${() => setActiveTab('home')}">Home</li><liattr.class.active="${is(activeTab, 'about')}"onclick="${() => setActiveTab('about')}">About</li><liattr.class.active="${is(activeTab, 'contact')}"onclick="${() => setActiveTab('contact')}">Contact</li></ul>${effect(activeTab, () => tabContent[activeTab()])}
`;app.render(document.body)

Effect format

The effect take comma-separated states and as the lastargument should be the action to take or the "thing" you need to bereturned.

effect(STATE, ...STATE, SIDE_EFFECT_ACTION)

effect(activeTab, currentPath, contentLoader)
